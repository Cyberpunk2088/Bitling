# Research principles for BITLING OMNI

The project studies public, general game-development principles rather than copying proprietary content, assets, characters, interfaces or source code.

## Adopted principles

- Accessibility is part of product quality, not an afterthought.
- Player intent should be easy to express even when the underlying simulation is complex.
- Audio, animation and UI feedback must reinforce the same action state.
- Animation uses blended states and one-shot reactions rather than isolated clips.
- Dialogue selection must use context, recency and personality to prevent repetition.
- Rendering quality must scale through explicit mobile and desktop performance tiers.
- Every feature is validated through functional, experience and polish passes.
